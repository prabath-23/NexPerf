# NexPerf v0.3.0

v0.3.0 moves NexPerf toward a configurable workstation observability platform.

## Focus

- Workspace navigation for Dashboard, Processes, Storage, Insights, History, NexPerf, and Settings
- Refined NexPerf identity with JetBrains Mono, a thunderbolt-inspired `N` mark, and generated favicon assets
- Config loading from `/etc/nexperf/config.toml` and `~/.config/nexperf/config.toml`
- UI-based config editing for polling, retention, refresh rate, storage scan limits, usage mode, and thresholds
- Storage intelligence for large folders and storage artifacts such as dependencies, caches, downloads, logs, Docker data, and generated outputs
- Process intelligence metadata including PPID, thread counts, runtime, categories, and process tree API data
- NexPerf self-observability for database size, sample counts, memory overhead, goroutines, active collectors, API timings, and uptime

## New APIs

```txt
GET /api/processes/detail?pid=<pid>
GET /api/processes/tree
GET /api/storage/summary
GET /api/config
GET /api/config/modes
PUT /api/config
GET /api/nexperf
```

## Notes

The v0.3.0 process tree is a first step toward deep process ancestry. It exposes parent PID and runtime metadata now, with room for nested trees and process detail pages in a later release.

Storage intelligence currently scans bounded directory levels and prioritizes practical workstation signals over exhaustive filesystem indexing.
